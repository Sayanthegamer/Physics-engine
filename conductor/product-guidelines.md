# Product Guidelines: GearEngine

## 1. Brand Identity & Voice
- **Tone:** Professional, precise, and empowering. Communication should be clear, technical yet accessible, avoiding unnecessary jargon.
- **Voice:** Confident and instructional. We are building a high-performance tool, and our language should reflect reliability and speed.
- **Visual Identity:** Clean, functional, and modern. Emphasize clarity over ornamentation. The engine must look state-of-the-art and visually stunning, avoiding outdated aesthetics from the 2000s.

## 2. User Experience (UX) Principles
- **Modern UI Aesthetics:** The user interface should feel premium, responsive, and alive. Utilize modern design principles (e.g., sleek themes, dynamic hover effects, smooth transitions) that wow the user while maintaining professional CAD-style interaction.
- **Performance First:** UI interactions must never block or stutter the core physics simulation. The experience should feel instantaneous, even on budget hardware.
- **CAD-Style Interaction:** Favor direct, spatial manipulation over complex menus. Users should interact with the 3D space intuitively (e.g., auto-snapping, drag-and-drop).
- **Graceful Degradation:** Features should scale elegantly based on hardware capabilities, ensuring a smooth baseline experience for all users.
- **Discoverability:** Complex mechanics should be easy to discover through contextual tooltips and immediate visual feedback during interaction.

## 3. Accessibility & Inclusivity
- Ensure strong color contrast for UI elements against the 3D viewport.
- Support standard keyboard shortcuts and accessible navigation for power users and those relying on keyboard input.
